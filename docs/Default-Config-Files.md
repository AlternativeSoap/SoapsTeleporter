# Default Config Files

These are the default config files generated by SoapsTeleporter on first run. You can copy these to reset a file or reference them when editing.

The full file has messages organized into these sections:
- **General** - Basic plugin messages
- **Self-Test** - Diagnostic messages
- **Portal Events** - Create, remove, connect, teleport messages
- **Economy** - Payment and cost messages
- **Errors** - Error messages
- **Permissions** - Permission denied messages
- **Wand** - Selection wand messages
- **Positions** - Position set/clear messages
- **Conditions** - All condition-related messages
- **RTP** - Random teleport messages
- **Protection** - Block protection messages
- **Linking** - Connection messages
- **Integrations** - Plugin integration messages
- **Commands** - Command usage and validation
- **Portal List** - Portal listing format
- **Portal Info** - Info display format
- **Help System** - Help command messages
- **Interactive Management** - GUI management messages

All messages support MiniMessage formatting with gradients, colors, hover text and click events.

## portals.yml

This file stores all your portal data. It starts empty:

```yaml
portals: {}
```

Portals are saved here automatically by the plugin. You generally don't need to edit this file manually.

## Customizing Messages

The messages file uses **MiniMessage** formatting. Here are some basics:

| Format | Example |
|--------|---------|
| Colors | `<red>`, `<green>`, `<#FF5733>` |
| Gradients | `<gradient:red:blue>text</gradient>` |
| Bold | `<bold>text</bold>` |
| Italic | `<italic>text</italic>` |
| Hover text | `<hover:show_text:'Tooltip'>text</hover>` |
| Click command | `<click:run_command:'/st help'>Click</click>` |

### Placeholders in Messages

| Placeholder | Used for |
|-------------|----------|
| `{portal}` | Portal name |
| `{player}` | Player name |
| `{cost}` | Cost amount |
| `{time}` | Time in seconds |
| `{world}` | World name |
| `{location}` | Location coordinates |
