V1.0.1 Changelog:

SoapsTeleporter 1.0.1 is a major internal architecture and systems update focused on performance, scalability, and long-term maintainability.

==================================================
NEW SYSTEMS
==================================================
- Added modular runtime service registry (`TeleporterRuntime`) for staged rewrite architecture.
- Added rolling runtime metrics registry for timing and counters.
- Introduced domain snapshot/mapping layer for portal definition, geometry, policy, and runtime state.

==================================================
RENDERING & PERFORMANCE
==================================================
- Replaced monolithic rendering flow with layered render engine foundation:
  - Frame layer
  - Interior layer
  - Ambient layer
  - Interaction layer hooks
- Added adaptive quality controller based on live load.
- Added visibility culling support (distance/FOV/line-of-sight checks).
- Added portal sleep controller for inactive portal rendering suppression.
- Added render event API hook: `PortalRenderEvent`.

==================================================
SHAPE PREVIEW & BUILDER WORKFLOW
==================================================
- Added shape engine presets:
  - rectangle, square, circle, oval, triangle, hexagon, octagon, random_rift
- Added shape preview session history with undo.
- Added shape transforms: rotate and mirror.
- Added shape validation service for loaded-chunk/world sanity checks.
- Added preview event API hook: `PortalPreviewEvent`.
- Added new command:
  - `/st shapepreview <preset|rotate|mirror|undo>`

==================================================
CONDITIONS ENGINE
==================================================
- Added expression parser/evaluator with nested logic:
  - `AND`, `OR`, `NOT`, and parenthesis grouping
- Added optional per-portal expression field (`conditionExpression`) with serialization.
- Added custom condition provider registration via public API.

==================================================
ROUTING & TELEPORT EXPANSION
==================================================
- Added multi-destination routing model with modes:
  - `FIXED`, `RANDOM`, `WEIGHTED`, `PERMISSION`
- Added per-portal linked destinations list serialization.
- Integrated routing resolution into teleport target selection.
- Added new commands:
  - `/st setrouting <portal> <fixed|random|weighted|permission>`
  - `/st setdestinations <portal> <dest1,dest2,...>`
- Added teleport prepare event API hook: `PortalTeleportPrepareEvent`.

==================================================
OBSERVABILITY & API
==================================================
- Added `/st debug overlay` runtime metrics view.
- Added lifecycle event hooks:
  - `PortalStabilizeEvent`
  - `PortalCollapseEvent`
- Added public API entry point:
  - `SoapsTeleporterApi`
  - `SoapsTeleporterApiProvider`

==================================================
GUI FRAMEWORK PROGRESSION
==================================================
- Extended GUI manager initialization to load YAML menu settings through SoapsCommon loader.
- Added YAML action bridge handlers (`prev_page`, `next_page`, `chat_input`) for migration compatibility.
- Added GUI shutdown lifecycle cleanup.

==================================================
DOCUMENTATION
==================================================
- Performed full Markdown documentation refresh across all `*.md` files in the module:
  - fixed invalid wiki links
  - aligned requirements and integration status with actual runtime behavior
  - documented 1.0.1 commands and capabilities
  - updated examples and routing/condition expression guidance
